# Гайд по решению lab4_var5-2

Файл решения: `main.cpp`.

Задача: найти все вхождения нескольких образцов в тексте. Образцы и текст состоят из чисел от `0` до `2^32 - 1`. Для этого используется алгоритм Ахо-Корасик.

## 1. Что делает программа целиком

Вход устроен так:

```text
образец 1
образец 2
...
образец k

текстовая строка 1
текстовая строка 2
...
```

Пустая строка отделяет образцы от текста.

Программа делает четыре основных шага:

1. Читает все образцы до пустой строки.
2. Строит trie по этим образцам.
3. Строит связи неудачи и выходные ссылки автомата Ахо-Корасика.
4. Читает текст число за числом и печатает найденные вхождения.

Формат ответа:

```text
строка, слово, номер_образца
```

`строка` и `слово` — позиция начала найденного образца в тексте.

## 2. Почему алфавит реализован правильно

В ТЗ алфавит — числа от `0` до `2^32 - 1`.

Поэтому переходы в trie хранятся так:

```cpp
std::unordered_map<std::uint32_t, std::size_t> next;
```

Ключ — не `char`, не строка, а именно `std::uint32_t`.

Это важно: если бы код искал по символам строки, то алфавит был бы меньше и решение не соответствовало бы варианту.

Пример:

```text
0002
02
2
```

Все эти записи после парсинга становятся одним числом `2`.

## 3. Вспомогательные структуры

### `Position`

```cpp
struct Position {
    std::size_t line;
    std::size_t word;
};
```

Хранит позицию числа в тексте:

- `line` — номер строки текста;
- `word` — номер числа в этой строке.

Нумерация начинается с `1`.

Строки с образцами не считаются строками текста.

### `PatternInfo`

```cpp
struct PatternInfo {
    std::size_t id;
    std::size_t length;
};
```

Хранит информацию об образце, который заканчивается в некоторой вершине автомата:

- `id` — порядковый номер образца во входе;
- `length` — длина образца в числах.

Длина нужна, чтобы восстановить позицию начала вхождения.

## 4. Вершина автомата

Внутри класса `AhoCorasick` есть структура `Node`:

```cpp
struct Node {
    std::unordered_map<std::uint32_t, std::size_t> next;
    std::size_t failure_link = 0;
    int output_link = -1;
    std::vector<PatternInfo> patterns;
};
```

Каждая вершина trie хранит:

- `next` — переходы по числам;
- `failure_link` — связь неудачи, то есть суффиксная ссылка в терминах Ахо-Корасика;
- `output_link` — ссылка на ближайшую вершину, где заканчивается образец;
- `patterns` — список образцов, которые заканчиваются прямо в этой вершине.

Вершины хранятся в массиве:

```cpp
std::vector<Node> nodes_ = std::vector<Node>(1);
```

`nodes_[0]` — корень trie.

Вместо указателей используются индексы вершин. Это проще для хранения и быстрее для копирования.

## 5. Построение trie: `AddPattern`

Функция:

```cpp
void AddPattern(const std::vector<std::uint32_t>& pattern, std::size_t pattern_id)
```

Добавляет один образец в trie.

Основная идея:

```cpp
std::size_t current = 0;
```

Начинаем из корня.

Дальше идём по всем числам образца:

```cpp
for (std::uint32_t value : pattern)
```

Для каждого числа проверяем, есть ли переход:

```cpp
auto found = nodes_[current].next.find(value);
```

Если перехода нет, создаём новую вершину:

```cpp
const std::size_t new_node = nodes_.size();
nodes_.push_back(Node());
nodes_[current].next[value] = new_node;
current = new_node;
```

Если переход уже есть, просто идём по нему:

```cpp
current = found->second;
```

Когда весь образец добавлен, в текущей вершине запоминаем, что здесь заканчивается образец:

```cpp
nodes_[current].patterns.push_back(PatternInfo{pattern_id, pattern.size()});
```

Также обновляем максимальную длину образца:

```cpp
if (pattern.size() > max_pattern_length_)
    max_pattern_length_ = pattern.size();
```

Она потом нужна для хранения последних позиций текста.

## 6. Связь неудачи `failure_link`

Связь неудачи, или `failure_link`, из вершины ведёт в вершину, которая соответствует самому длинному собственному суффиксу текущей строки.

Например, если вершина соответствует последовательности:

```text
1 2 3 4
```

то `failure_link` может вести в вершину:

```text
2 3 4
```

если такая последовательность есть в trie.

Если подходящего суффикса нет, ссылка ведёт в корень.

Зачем это нужно:

когда очередное число не подходит для текущего пути, мы не начинаем поиск заново, а переходим по связи неудачи и продолжаем.

## 7. Выходная ссылка

`output_link` ведёт к ближайшей вершине по цепочке связей неудачи, где заканчивается какой-нибудь образец.

Это нужно для случаев, когда один образец является суффиксом другого.

Пример образцов:

```text
1 2 3
2 3
3
```

Когда автомат прочитал `1 2 3`, он должен найти не только `1 2 3`, но ещё и `2 3`, и `3`.

Без выходных ссылок пришлось бы каждый раз идти по всей цепочке связей неудачи. С `output_link` мы быстро прыгаем только по полезным вершинам.

## 8. Построение ссылок: `BuildLinks`

Функция:

```cpp
void BuildLinks()
```

Строит связи неудачи и выходные ссылки обходом в ширину.

Для обхода используется очередь:

```cpp
std::queue<std::size_t> queue;
```

Сначала дети корня получают связь неудачи в корень:

```cpp
for (const auto& edge : nodes_[0].next) {
    const std::size_t child = edge.second;
    nodes_[child].failure_link = 0;
    nodes_[child].output_link = -1;
    queue.push(child);
}
```

Потом берём вершины из очереди:

```cpp
while (!queue.empty())
```

Для каждого перехода `vertex --value--> child` нужно найти связь неудачи для `child`.

Берём связь неудачи родителя:

```cpp
std::size_t fallback = nodes_[vertex].failure_link;
```

И поднимаемся по связям неудачи, пока не найдём вершину, из которой есть переход по `value`:

```cpp
while (fallback != 0 && nodes_[fallback].next.find(value) == nodes_[fallback].next.end())
    fallback = nodes_[fallback].failure_link;
```

Если такой переход нашёлся, связь неудачи ребёнка ведёт туда:

```cpp
auto transition = nodes_[fallback].next.find(value);
if (transition != nodes_[fallback].next.end() && transition->second != child)
    nodes_[child].failure_link = transition->second;
else
    nodes_[child].failure_link = 0;
```

Проверка `transition->second != child` нужна для защиты случая с детьми корня.

После этого строится `output_link`:

```cpp
const std::size_t suffix = nodes_[child].failure_link;
if (!nodes_[suffix].patterns.empty())
    nodes_[child].output_link = static_cast<int>(suffix);
else
    nodes_[child].output_link = nodes_[suffix].output_link;
```

То есть:

- если по связи неудачи сразу есть образец, выходная ссылка ведёт туда;
- иначе берём выходную ссылку вершины по связи неудачи.

## 9. Переход автомата: `NextState`

Функция:

```cpp
std::size_t NextState(std::size_t state, std::uint32_t value) const
```

Принимает:

- `state` — текущую вершину автомата;
- `value` — очередное число из текста.

Возвращает новую вершину автомата.

Если из текущей вершины нет перехода по `value`, идём по связям неудачи:

```cpp
while (state != 0 && nodes_[state].next.find(value) == nodes_[state].next.end())
    state = nodes_[state].failure_link;
```

Если переход нашёлся, идём по нему:

```cpp
auto transition = nodes_[state].next.find(value);
if (transition != nodes_[state].next.end())
    return transition->second;
```

Если даже из корня перехода нет, остаёмся в корне:

```cpp
return 0;
```

## 10. Поиск найденных образцов: `ForEachMatch`

Функция:

```cpp
template <typename Callback>
void ForEachMatch(std::size_t state, Callback on_match) const
```

Она смотрит, какие образцы заканчиваются в текущем состоянии.

Сначала проверяется сама вершина:

```cpp
for (const PatternInfo& pattern : nodes_[current].patterns)
    on_match(pattern);
```

Потом идём по `output_link`:

```cpp
current = nodes_[current].output_link;
```

Так находятся все образцы, которые являются суффиксами текущей прочитанной последовательности.

`Callback` здесь — действие, которое нужно выполнить для каждого найденного образца.

В `main` этим действием является печать ответа.

## 11. Парсинг чисел: `ParseNumbers`

Функция:

```cpp
template <typename Callback>
void ParseNumbers(const std::string& line, Callback on_number)
```

Она получает строку и вызывает `on_number(value)` для каждого числа в строке.

Пример:

```text
01 2 0003
```

Будут вызовы:

```cpp
on_number(1);
on_number(2);
on_number(3);
```

Сначала пропускаются пробелы:

```cpp
while (index < line.size() &&
       std::isspace(static_cast<unsigned char>(line[index])))
    ++index;
```

Потом читается число:

```cpp
std::uint64_t value = 0;
while (index < line.size() &&
       !std::isspace(static_cast<unsigned char>(line[index]))) {
    value = value * 10 + static_cast<std::uint64_t>(line[index] - '0');
    ++index;
}
```

После этого число передаётся наружу:

```cpp
on_number(static_cast<std::uint32_t>(value));
```

Почему сначала `std::uint64_t`, а потом `std::uint32_t`:

- входное число может быть до `4294967295`;
- при накоплении десятичных цифр безопаснее использовать тип шире;
- в автомат передаём уже значение алфавита `uint32_t`.

## 12. Чтение образцов в `main`

Сначала создаётся автомат:

```cpp
AhoCorasick automaton;
```

`pattern_id` — номер текущего образца:

```cpp
std::size_t pattern_id = 1;
```

Дальше читаем строки до пустой:

```cpp
while (std::getline(std::cin, line)) {
```

Каждая непустая строка превращается в вектор чисел:

```cpp
std::vector<std::uint32_t> pattern;
ParseNumbers(line, [&pattern](std::uint32_t value) {
    pattern.push_back(value);
});
```

Здесь используется лямбда:

```cpp
[&pattern](std::uint32_t value) {
    pattern.push_back(value);
}
```

Она означает: для каждого найденного числа добавить его в `pattern`.

Если строка пустая, образцы закончились:

```cpp
if (pattern.empty())
    break;
```

Иначе добавляем образец в автомат:

```cpp
automaton.AddPattern(pattern, pattern_id);
++pattern_id;
```

После чтения всех образцов строим ссылки:

```cpp
automaton.BuildLinks();
```

## 13. Зачем нужен `recent_positions`

Поиск идёт слева направо. Когда автомат находит образец, он находит его **конец**.

Но в ответ нужно вывести позицию **начала**.

Например, текст:

```text
7 8 9 10
```

Образец:

```text
8 9 10
```

Когда автомат прочитает `10`, он поймёт, что образец найден. Но начало было на числе `8`.

Поэтому хранятся позиции последних чисел:

```cpp
std::deque<Position> recent_positions;
```

Если максимальная длина образца равна `max_pattern_length`, то больше позиций хранить не нужно:

```cpp
if (recent_positions.size() > max_pattern_length)
    recent_positions.pop_front();
```

Для образца длины `pattern.length` позиция начала:

```cpp
const std::size_t start_index = recent_positions.size() - pattern.length;
const Position& start = recent_positions[start_index];
```

## 14. Чтение текста и поиск

После пустой строки начинается текст:

```cpp
while (std::getline(std::cin, line)) {
    ++line_number;
    std::size_t word_number = 0;
```

Каждая новая строка текста увеличивает `line_number`.

`word_number` начинается с нуля для каждой строки.

Дальше строка парсится на числа:

```cpp
ParseNumbers(line, [&](std::uint32_t value) {
```

Для каждого числа:

1. Увеличиваем номер слова:

```cpp
++word_number;
```

2. Запоминаем позицию числа:

```cpp
recent_positions.push_back(Position{line_number, word_number});
```

3. Оставляем только последние `max_pattern_length` позиций:

```cpp
if (recent_positions.size() > max_pattern_length)
    recent_positions.pop_front();
```

4. Делаем переход автомата:

```cpp
state = automaton.NextState(state, value);
```

5. Проверяем все образцы, которые закончились здесь:

```cpp
automaton.ForEachMatch(state, [&](const PatternInfo& pattern) {
```

6. Вычисляем позицию начала найденного образца:

```cpp
const std::size_t start_index = recent_positions.size() - pattern.length;
const Position& start = recent_positions[start_index];
```

7. Печатаем ответ:

```cpp
std::cout << start.line << ", " << start.word << ", " << pattern.id << '\n';
```

## 15. Мини-пример работы автомата

Пусть образцы:

```text
1 2
2
```

Текст:

```text
1 2
```

После чтения `1` автомат находится в вершине `1`. Совпадений нет.

После чтения `2` автомат находится в вершине `1 2`.

В этой вершине заканчивается образец `1 2`, поэтому печатаем:

```text
1, 1, 1
```

Но у вершины `1 2` есть суффикс `2`, а `2` тоже образец. Через `output_link` находится второй образец:

```text
1, 2, 2
```

## 16. Сложность

Пусть:

- `P` — суммарная длина всех образцов;
- `T` — количество чисел в тексте;
- `Ans` — количество найденных вхождений.

Построение trie:

```text
O(P)
```

Построение ссылок:

```text
O(P * средняя стоимость поиска в unordered_map)
```

На практике это близко к `O(P)`.

Поиск:

```text
O(T + Ans)
```

Память:

```text
O(P)
```

Плюс `recent_positions`, который хранит только `max_pattern_length` позиций.

## 17. Что важно уметь объяснить на защите

Коротко:

1. Trie хранит все образцы как пути из корня.
2. Суффиксная ссылка показывает, куда перейти при несовпадении.
3. Выходная ссылка помогает найти образцы, являющиеся суффиксами текущего состояния.
4. Автомат читает текст один раз слева направо.
5. Позиция начала находится через длину образца и очередь последних позиций.
6. Алфавит реализован как `uint32_t`, поэтому числа не превращаются в символы.

## 18. Частые вопросы

### Почему нельзя хранить переходы в массиве?

Алфавит имеет размер `2^32`. Массив такого размера в каждой вершине невозможен по памяти.

Поэтому используется:

```cpp
std::unordered_map<std::uint32_t, std::size_t>
```

В ней хранятся только реально существующие переходы.

### Почему порядок вывода может отличаться от примера?

В условии сказано:

```text
Порядок следования вхождений образцов несущественен.
```

Программа печатает совпадения в момент, когда автомат их находит.

### Почему текст может продолжаться через строки?

Алгоритм ищет по потоку чисел. Перевод строки нужен только для правильного вывода позиции.

Если образец начался в конце одной строки и закончился в начале другой, он тоже будет найден.

Это соответствует формулировке про текст из слов или чисел и отсутствие ограничений на длину строк.

### Почему пустая строка в тексте не ломает поиск?

Пустая строка текста просто не содержит чисел.

`line_number` всё равно увеличивается, потому что это реальная строка текста.

Состояние автомата не сбрасывается, потому что текст рассматривается как последовательность чисел.
